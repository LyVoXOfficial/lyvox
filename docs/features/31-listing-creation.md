## 1. Зачем это (Problem & Outcome)
- **Проблема.** Создание объявления — узкое место предложения (ликвидность). Должно быть быстрым, но со структурой (категорийные поля, медиа, валидации) и премодерацией.
- **Исход.** Продавец создаёт объявление без ручной помощи; обязательные поля по категории; медиа со статусом модерации; автосейв.
- **KPI:** конверсия «начал → опубликовал»; время создания; доля черновиков, дошедших до публикации.

## 2. Контекст и текущее состояние (As-is)
- 8-шаговый мастер (`app/post`), автосейв, категорийные поля, каскады (марка/модель/год для транспорта), медиа (`upload-gallery`, `/api/media/*` signed storage, reorder, статус модерации).
- Каталог-схема (`/api/catalog/*`, `catalog_schema_tables`).
- Известно (из аудита): `prompt()`-OTP в форме надо заменить на нормальный компонент; кэшировать catalog schema; `<img>` без srcSet/skeleton.

## 3. Пользователи и сценарии
- **Registered/Verified seller:** «Создать объявление пошагово, с фото и нужными полями».
- **Business/trader:** доп-поля и гейты ([[14-consumer-trader-rights]], [[15-recupel-dac7-compliance]]).

## 4. UX и поведение (детально)
- Шаги: категория → базовые поля (заголовок/описание/цена/валюта/локация/состояние/контакт-политика) → категорийные поля → медиа → проверка телефона → превью → публикация.
- **Автосейв/черновик**; возврат к черновику.
- **Медиа:** загрузка, переупорядочивание, статус модерации; (улучшение) LQIP/skeleton, srcSet.
- **AI-хелпер (опц., roadmap [[50-ai-listing-from-photo]]):** заполнение из фото, но никогда авто-публикация без подтверждения.
- Состояния: draft / на модерации / опубликовано / отклонено (с апелляцией) / неактивно.

## 5. Логика и правила
- Валидации базовых и категорийных полей; бельгийские валидаторы (индекс, VAT/CBE, IBAN, IMEI — `belgium_validation_functions`).
- **Гейты публикации:** премодерация (AI + fraud-rules [[38-moderation-reports]]); Recupel для профи-электроники ([[15-recupel-dac7-compliance]]); trader-раскрытие ([[14-consumer-trader-rights]]).
- Краевые: незаполненные обяз-поля, отклонение модерацией, истечение черновика.

## 6. Данные (Data model)
- `adverts` (+ category-specific таблицы), `media` (статус модерации), каталог-схема. `seller_type`/`advert_business` для бизнес-листингов.
- RLS: владелец пишет своё; публичное чтение опубликованных. Индексы поиска ([[33-search]]).

## 7. API / интеграции
- `/api/adverts` (CRUD), `/api/media/*` (sign/complete/list/reorder), `/api/catalog/*`, `/api/categories/tree`.
- Хук фото-верификации ([[17-image-verification]]) на `/api/media/complete`.

## 8. Безопасность, приватность, комплаенс
- Премодерация до публикации; signed storage; гейты Recupel/trader; DSA (премодерация + апелляции).

## 9. Аналитика
- Воронка шагов мастера; drop-off по шагам; время до публикации; доля отклонённых.

## 10. Доступность и i18n
- Доступные формы/шаги; i18n всех строк (заменить хардкод по аудиту); категорийные словари локализованы ([[42-i18n-localization]]).

## 11. Тестирование
- Unit: валидаторы (VAT/CBE/IBAN/IMEI), каскады. API: создание/редактирование, медиа. e2e: полный мастер до публикации; отклонение модерацией.

## 12. План внедрения (Rollout)
- **Поддержка/улучшения:** заменить `prompt()`-OTP на компонент; кэш catalog schema; LQIP/skeleton; хук фото-верификации. Усилие: S–M.

## 13. Открытые вопросы и решения
- Длительность жизни черновика. Какие категорийные поля обязательны в MVP-категориях.

## 14. Критерии готовности (DoD)
- [ ] Полный мастер с автосейвом и категорийными полями работает.
- [ ] Гейты публикации (модерация/Recupel/trader) применяются.
- [ ] Медиа со статусом модерации + улучшения загрузки.
- [ ] i18n без хардкода; тесты §11 зелёные.

## Ревью-требования и sign-off (2026-06-28)

### UX (БЛОКЕР — воронка)
- **Двойной барьер «логин → обязательная верификация ДО формы» = убийца supply на cold-start.** Переписать §4: разрешить заполнять и **сохранять черновик без верификации**; жёсткий гейт верификации — только на шаге «Публикация» (эффект Зейгарника: барьер в конце, когда вложен труд, конвертит лучше). **P0-UX.**
- **Адаптивное число шагов по сложности категории:** простые (мода/дом/даром) — 3–4 шага, тяжёлые (транспорт/недвижимость) — полный набор; необязательное — под «Дополнительно» (смыкается с [[62-product-category-detail]]).
- **Прогресс ценности:** прогресс-бар + «Ещё 2 шага до публикации» + превью-тизер (снижает drop-off).
- **Локация обязательна** на шаге базовых полей (гео-автоподстановка); без неё не публиковать.
- **Поток «отклонено модерацией» — эмпатичный и actionable:** причина + что исправить + «Подать на пересмотр» (копирайт + UX, не только перечисление).
- **`prompt()`-OTP** заменить на доступный модальный компонент — в DoD.
- **a11y форм (WCAG 2.1 AA):** каждый шаг `fieldset/legend`+фокус на заголовок; видимые `<label>` (не placeholder); ошибки `aria-describedby`+`aria-invalid`+фокус на первую; OTP-модалка фокус-trap/Esc; reorder медиа с клавиатуры (кнопки «Вверх/Вниз»); статус модерации медиа `aria-live`; степпер `aria-current="step"`; тач-таргеты ≥44px.

### DEV (фундаменты F9, S5)
- **fail-open на `POST /api/adverts:31` подтверждён** — привести к `checkUserBlocked(user.id, { failClosed: true })` (сквозное S5) в §5/§8.
- **fraud-detection не вызывается в рантайме (F9):** «премодерация (AI + fraud-rules)» декларирована, но Edge Function `fraud-detection` имеет 0 call-sites. Добавить вызов на create/publish (синхронно или async со статусом `pending`), записать `account_flags`/score, гейтить публикацию (связь с [[38-moderation-reports]]).
- **Каскад year vs generation:** встроить generation-чузер из [[62-product-category-detail]] в шаг категорийных полей транспорта (сейчас generation не выбирается вовсе).
- Тесты: create при заблокированном юзере (fail-closed); премодерация-гейт; срабатывание fraud-rule.

### SEO (фундамент F12)
- **SEO-валидации title:** 10–70 симв., подсказка «бренд + модель + ключ. характеристика»; запрет CAPS/эмодзи-спама (мусорный заголовок → мусорный slug).
- **Описание:** минимум ~120 симв.; анти-дубль-чек (одинаковый текст по N объявлениям → предупреждение); подсказки BE-ключей NL/FR (§0.3 SEO-ревью).
- **Публичность медиа:** различить приватные signed (UI-галерея) и **публичный превью-вариант** для og:image/JSON-LD/`next/image` (rich results питаются отсюда → пометить категорийные поля «влияют на видимость в Google»).
- **`content_locale`** объявления сохранять при создании (для `inLanguage`/hreflang/OG).
- **slug-301:** при смене заголовка старый slug → 301 на новый; canonical на id.
- **Вклад в sitemap:** `status=active` → добавляется, снятие → удаляется.

**Вердикт:** ⛔ Блокер по UX (упущенная воронка) — пока §4 не перепишет черновик-до-верификации + адаптивные шаги + поток отклонения и не добавит a11y форм, sign-off нельзя. DEV/SEO — правки уровня S–M. После правок → 🔄/✅.
