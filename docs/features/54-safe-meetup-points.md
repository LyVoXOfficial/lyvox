# PRD: Безопасные точки встречи + QR/NFC подтверждение

> **Статус кода:** ⛔ НЕ НАЧАТО
> **Категория:** Бонус
> **Приоритет:** P2
> **Зависит от:** [[10-escrow-safe-deal]] (in-person escrow — отложен), [[11-disputes]]
> **Последнее обновление:** 2026-06-27

---

## 1. Зачем это (Problem & Outcome)
- **Проблема.** Локальный самовывоз — доминирующий режим в BE и самый незащищённый (escrow на in-person отложен как спорный).
- **Исход.** Каталог безопасных точек встречи (напр. у полицейских участков/публичных мест) + QR/NFC-подтверждение передачи → доказательство для in-person сделок без сложного escrow.
- **KPI:** доля локальных сделок через safe-meetup; снижение инцидентов на самовывозе.

## 2. Контекст и текущее состояние (As-is)
- Нет. In-person escrow осознанно отложен ([[10-escrow-safe-deal]] §12 LATER).

## 3. Пользователи и сценарии
- «Договориться встретиться в безопасной точке»; «подтвердить передачу сканом QR/NFC».

## 4. UX и поведение (детально)
- Выбор точки встречи из каталога (карта/список); назначение времени.
- На встрече — обе стороны сканируют QR/тапают NFC → фиксируется подтверждение передачи (доказательство).
- Состояния: назначено/подтверждено/не состоялось.

## 5. Логика и правила
- Подтверждение передачи = событие-доказательство ([[11-disputes]]).
- (Если совмещать с escrow позже) release по подтверждению на верифицированной точке.
- Краевые: не явились, спор о состоянии товара на месте.

## 6. Данные (Data model)
- `meetup_points { id, name, address, type, geo }` (каталог), `meetup_confirmations { id, deal_id/advert_id, point_id, confirmed_by, ts }`. RLS участники.

## 7. API / интеграции
- `/api/meetups/points`, `/api/meetups/confirm`; карты.

## 8. Безопасность, приватность, комплаенс
- Минимизация геоданных; добровольность; не гарантия безопасности (дисклеймер).

## 9. Аналитика
- Использование точек, доля подтверждённых встреч, инциденты.

## 10. Доступность и i18n
- Доступный выбор точек; i18n `meetup.*` (NL/FR/EN).

## 11. Тестирование
- API: точки/подтверждение. e2e: назначить встречу → подтвердить QR.

## 12. План внедрения (Rollout)
- **LATER:** после проверенных dispute-ops. Усилие: M.

## 13. Открытые вопросы и решения
- Источник каталога точек (договорённости с локациями). Связывать ли с escrow.

## 14. Критерии готовности (DoD)
- [ ] Каталог точек + назначение встречи.
- [ ] QR/NFC-подтверждение как доказательство.
- [ ] i18n; тесты §11 зелёные.

## Ревью-требования и sign-off (2026-06-28)

> P2/бонус, не срочно, но при реализации liability-формулировки и согласие локаций обязательны. Применить сквозной F-фундамент F4 (GDPR: основание/retention/DSAR).

### LEGAL
- **Не называть точки «безопасными» (ключевое, анти-liability/UCPD).** «Каталог безопасных точек» создаёт **representation of safety** → при инциденте на/у точки LyVoX рискует ответственностью за misleading assurance. Переименовать в **«suggested public meeting points»** + явный дисклеймер (Terms + UI): «LyVoX не гарантирует безопасность; точки предложены информационно; следуйте здравому смыслу и местным правилам». Заменить формулировку §1/§4 «безопасные точки».
- **Согласие локаций — особенно полиция.** Нельзя указывать конкретные публичные/госучреждения («у полицейских участков») как «партнёрские» **без их письменного согласия**; ассоциация бренда с полицией без разрешения = риск претензии от властей. Только реально согласованные точки помечать партнёрскими; остальные — как generic public spaces. Закрыть открытый вопрос §13 (источник каталога).
- **GDPR геоданных + QR/NFC (F4).** Геолокация точки + время + кто подтвердил = ПДн о перемещениях/контактах. Зафиксировать правовое основание (Art.6(1)(b) — доказательство сделки), retention confirmations, **отсутствие background-tracking** (только явный скан, явный consent на скан). Минимизация (есть в §8 — конкретизировать).
- **QR/NFC = доказательство встречи, не состояния товара.** В dispute-контексте (PRD 11) не оверселлить юридическую силу: подтверждает факт передачи, не качество/комплектность.

**Вердикт:** 🔄 — при реализации обязательны: переименование «safe» → «suggested public meeting points» + safety-дисклеймер, согласие локаций (запрет ассоциации с полицией без разрешения), GDPR-минимизация геоданных. С ними → ✅.
