## 1. Зачем это (Problem & Outcome)
- **Проблема.** Поиск — основной способ найти товар; нулевой результат — главный триггер оттока.
- **Исход.** Быстрый FTS + фильтры/сортировки/гео/пагинация, понятный empty-state с подсказками и «сохранить поиск».
- **KPI:** доля поисков с результатом; CTR из выдачи; доля zero-result (снижать); сохранённые поиски.

## 2. Контекст и текущее состояние (As-is)
- Postgres FTS, фильтры, гео, сортировки, пагинация (`search_function`, `search_adverts*`, `/api/search`, `SearchBar`, `SearchFilters`).
- Проекция на карточные поля + отдельный rate-limit bucket для instant-поиска (недавний коммит).
- `verified_only`-фильтр, фильтр по состоянию (`search_adverts_condition`).
- Известно: нет явного empty-state/«save search» на `search/page.tsx` (улучшение → [[34-discovery-extras]]); бусты пока не влияют на ранжирование (см. ниже).

## 3. Пользователи и сценарии
- **Покупатель:** «Найти по запросу/категории, отфильтровать по цене/локации/состоянию/верифицированным, отсортировать».
- **Покупатель без результата:** «Получить подсказки и сохранить поиск с алертом».

## 4. UX и поведение (детально)
- Строка поиска + фильтры (категория, цена, локация, состояние, верификация, доступность safe-deal — когда появится).
- Сортировки: релевантность, новые, цена, проверенные продавцы.
- Состояния: loading, результаты, **empty-state** (подсказки + «сохранить поиск»), error.
- Пагинация/бесконечная лента.

## 5. Логика и правила
- FTS по заголовку/описанию + атрибут-фильтры; гео-радиус; bounded/paginated запросы.
- **Бусты в ранжировании:** benefits покупаемы, но `search_adverts` их игнорирует — включить влияние **только при наличии ликвидности** ([[39-monetization-billing]]).
- Краевые: пустой запрос, слишком широкий/узкий фильтр, опечатки (roadmap: подсказки).

## 6. Данные (Data model)
- `adverts` + индексы FTS/атрибутов; RPC `search_adverts`. Публичное чтение опубликованных.

## 7. API / интеграции
- `GET /api/search` (карточная проекция), `/api/top-adverts`, `/api/top-sellers`. Rate-limit instant-bucket.

## 8. Безопасность, приватность, комплаенс
- Только опубликованные/не-скрытые листинги; учёт mute/hidden ([[01-discover-swipe-system]]); rate-limit от абуза.

## 9. Аналитика
- Запросы, zero-result rate, CTR выдачи, использование фильтров/сортировок.

## 10. Доступность и i18n
- Доступные фильтры/пагинация; i18n `search.*` (NL/FR/EN); hreflang.

## 11. Тестирование
- Unit: построение запроса/фильтров. API: search с фильтрами/сортировками/пагинацией. e2e: поиск → выдача → карточка; zero-result → подсказки.

## 12. План внедрения (Rollout)

## Ревью-требования и sign-off (2026-06-28)

### SEO (БЛОКЕР — фундамент F12)
- **Индексируемость не задана.** Параметрический поиск (категория × цена × локация × состояние × сортировка) = бесконечная краулер-ловушка + дубль-контент. Явно: `/search` и `/discover` → `robots: noindex, follow`; убрать из sitemap; `disallow: /search` в `robots.ts`.
- **`search/page.tsx` — `"use client"`** (подтверждено): из клиентского компонента `generateMetadata`/`robots` невозможны → `noindex` физически не выставить. **Acceptance:** разбить на серверный `page.tsx` (`export const metadata = { robots:{ index:false } }`) + клиентский дочерний компонент.
- Различить **search (noindex)** vs **категорийные листинги `/c/*` (index, владеет [[32-category-catalogs]])** — поиск не порождает индексируемые URL.
- **WebSite + `SearchAction`:** home должна отдавать `SearchAction` с `urlTemplate` `/search?q={search_term_string}` (sitelinks searchbox) — зона ответственности этого PRD (owner: search).
- Убрать декларацию «hreflang выдачи» из §10 — на `noindex`-странице hreflang бессмыслен.
- Внутренняя перелинковка из zero-result/empty-state на популярные `/c/*` (UX + SEO-вес).

### UX (фундаменты — единые правила зоны)
- **Скелетон вместо спиннера:** подключить `AdsGridSkeleton` (импортирован, но не используется) — требование, не «улучшение».
- **Одна модель фильтров:** desktop — мгновенно (живая выдача без кнопки); mobile — нижний лист с кнопкой «Показать N результатов» (динамическое число).
- **Единая пагинация:** `IntersectionObserver` + sentinel (убрать хрупкий `setTimeout(2000)`); стабильная сортировка с `id` последним ключом (иначе keyset/offset даёт дубли/пропуски); desktop — «Показать ещё».
- **Zero-result как retention:** обязательный empty-state с «🔔 Сохранить поиск, пришлём новинки» + «Расширить радиус»/«Очистить фильтры».
- **Анти-социальное доказательство:** при `total < N` не выпячивать число («Все объявления по запросу» вместо «3 объявления»); упор на качество («Только проверенные продавцы»).
- **a11y:** `aria-live="polite"` («Найдено N»); фильтры-Sheet — фокус-trap/Esc/возврат фокуса; слайдер цены — клавиатура + `aria-valuenow/min/max`; чипы фильтров — кнопки «Убрать {фильтр}»; тумблер «Только верифицированные» — `role=switch`; «Показать ещё» как клавиатурная альтернатива инфинит-скроллу; скелетоны `aria-busy`; тач-таргеты ≥44px.

### DEV (фундамент F8)
- **Формула буст-ранжирования** (детерминированная, за флагом `boost_ranking_enabled`): `final = base_relevance + min(boost_cap, boost_weight * is_boosted)` — аддитивный bounded бонус (не множитель), чтобы нерелевантный буст не топил выдачу; тай-брейкеры детерминированы.
- **`getClientIp` доверяет `x-forwarded-for` (A-7, F8):** instant-bucket per-IP rate-limit обходится спуфингом — починить (доверять только последнему доверенному hop).

### DATA (фундамент F6)
- **Релевантность FTS настроить и измерять:** `setweight(title,'A') || setweight(description,'B')`; `ts_rank_cd` с нормализацией по длине; словари `dutch`/`french`/`english` (i18n).
- **Search-события (единый sink `analytics_events`, F6):** `search_performed {query_hash, filters, result_count, zero_result:bool, sort}` + `search_result_click {query_hash, advert_id, position}` — закрывает KPI zero-result-rate и даёт сырьё для MRR/CTR@k.
- **Формула буст-микса с cap** (см. DEV выше) + zero-result loop (failed queries → словарь синонимов/редиректов).
- **Дедуп `advert_views` (F11):** ранжирование тянет `top_sellers.avg_views` из недедупленного `advert_views` (накрутка → leakage в выдачу) — закрыть общим фиксом дедупа.

**Вердикт:** ⛔ Блокер по SEO (краулер-ловушка + client-component блокирует `noindex`). UX/DEV/DATA — доработки. После server/client split + `noindex` + правок → 🔄/✅.
