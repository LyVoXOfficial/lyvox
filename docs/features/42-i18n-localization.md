## 1. Зачем это (Problem & Outcome)
- **Проблема.** Бельгия многоязычна (NL/FR + EN); качество NL/FR/EN критично для доверия и SEO. Полу-переведённые языки вредят бренду.
- **Исход.** Первоклассные NL/FR/EN (DE/RU — опц. интерфейсные), без хардкода, с hreflang/SEO и локализованными словарями/шаблонами.
- **KPI:** покрытие ключей по локалям; органический трафик по NL/FR-запросам; доля пользователей на родном языке.

## 2. Контекст и текущее состояние (As-is)
- 5 локалей синхронизированы построчно (`i18n/locales/{nl,fr,en,de,ru}.json` — по 1887 строк), `LanguageSwitcher`, `lib/i18n`, `i18n/server`.
- SEO: canonical/OG (`lib/seo`); недавний фикс canonical `/ad/...`.
- Известно (TODO/strategy): остаточный хардкод (home hero, login, onboarding `ru: englishMessages` — мог быть починен; проверить), hreflang/alternate OG неполны, словари vehicle i18n из curated CSV, de-rollout QA.

## 3. Пользователи и сценарии
- «Пользоваться на NL/FR/EN»; «получать письма/уведомления на своём языке»; поисковики — корректные hreflang.

## 4. UX и поведение (детально)
- Переключатель языка (запоминать выбор; в футере и настройках).
- Локаль пользователя хранится и используется для UI/уведомлений.
- Локализованные словари категорий (имена собственные — curated, не машинный перевод).
- hreflang + per-locale OG/alternateLocale.

## 5. Логика и правила
- Все публичные строки через `t()`; запрет хардкода (особенно RU-only в проде).
- Фоллбэк на EN при отсутствии ключа; юр/compliance тексты — выверенные NL/FR/EN сперва.
- Краевые: отсутствующий ключ, неполный перевод словаря, новые языки.

## 6. Данные (Data model)
- Локали-JSON; i18n-таблицы словарей (vehicle make/model/generation, и т.д.); locale в `profiles`.

## 7. API / интеграции
- `/api/locale`; join словарей по locale; SEO-хелперы (hreflang/OG).

## 8. Безопасность, приватность, комплаенс
- Юр-тексты per-locale корректны (GDPR/DSA/consumer). Без утечки непереведённого чувствительного.

## 9. Аналитика
- Распределение по локалям, покрытие ключей, fallback-частота, SEO по NL/FR.

## 10. Доступность и i18n
- lang-атрибуты, направление, форматы чисел/дат/валют per-locale; доступный переключатель.

## 11. Тестирование
- Unit: загрузка локалей, fallback, форматтеры. e2e: смена языка сохраняется; ключевые экраны без хардкода; hreflang присутствует.

## Ревью-требования и sign-off (2026-06-28)

### SEO (фундамент F12 — владелец hreflang/og:locale)
- **hreflang только декларирован, не специфицирован.** Аудит 02 нашёл: домен hardcoded `lyvox.be` (без www) → mismatch с `www.lyvox.be`; нет `x-default`; alternates только на home, не per-page; `?lang=` слабее path-based.
- **Формат тегов — региональные:** `nl-BE`/`fr-BE` (не `nl`/`fr`!) + `en`, `de-DE`, `ru` + `x-default → EN`. Для BE критично (один язык в разных странах — регион различает).
- **Хелпер `buildAlternates(pathname)`:** per-page languages + `x-default`, домен через `getSiteUrl()` (www).
- **Маппинг `ogLocale`** (этот PRD — владелец): `en→en_US, nl→nl_BE, fr→fr_BE, de→de_DE, ru→ru_RU` + `og:locale:alternate` со всеми остальными. Аудит: сейчас `og:locale=ru` при cookie — починить.
- **Дубль-контент при `?lang=`:** связка `canonical ↔ hreflang` (5 языковых версий одного URL без неё = каннибализация).
- **Бельгийские ключевые токены NL/FR** (§0.3 SEO-ревью) — в i18n-словарь как переиспользуемые токены для SEO-шаблонов title/description/H1.
- **Acceptance ужесточить:** §11 e2e «hreflang присутствует» → «hreflang **корректен**: www-домен, region-tags, x-default, reciprocal, self-canonical».
- **Решение по path-based locale:** рекомендация перейти на `/{locale}/` сегмент (среднесрочно) для чистых per-locale индексируемых URL; на старте — минимум корректный `?lang=` + canonical/hreflang. Зафиксировать как открытый вопрос с рекомендацией.

### UX (сквозное правило зоны)
- Все RU-строки ревью = эталон смысла → ключ i18n → NL/FR/EN (приоритет) + DE/RU. Учесть расширение текста (FR/DE на 20–30% длиннее — штампы/чипы/кнопки тестировать на переполнение). Юр-термины и имена собственные (Car-Pass/EPC/Recupel/itsme) — curated, не машинный перевод. Локализованные ссылки terms/privacy per locale.
