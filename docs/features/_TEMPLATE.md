## 1. Зачем это (Problem & Outcome)
- **Проблема пользователя / бизнеса:** что болит сейчас.
- **Какой исход хотим:** измеримый результат.
- **Связь со стратегией LyVoX:** trust-first / ликвидность / комплаенс / монетизация.
- **Метрики успеха (KPI):** 1–3 цифры, как поймём, что сработало.

## 2. Контекст и текущее состояние (As-is)
- Что уже есть в коде (файлы/таблицы/эндпойнты — со ссылками на реальные пути).
- Что работает, что сломано/не подключено.
- Известные ограничения.

## 3. Пользователи и сценарии (User stories)
- Роли: Guest / Registered / Verified seller / Business / Moderator / Admin / Support.
- Истории: «Как <роль>, я хочу <действие>, чтобы <ценность>».
- Главные happy-path сценарии + альтернативные.

## 4. UX и поведение (детально)
- Экран(ы), вход/выход, навигация.
- Состояния: loading / empty / error / success / disabled / offline.
- Все интеракции по шагам (включая жесты/анимации, если применимо).
- Копирайт ключевых строк (NL/FR/EN — минимум EN-плейсхолдеры).
- Мобайл vs десктоп различия.

## 5. Логика и правила (Business logic)
- Алгоритмы, формулы, пороги, лимиты.
- Валидации (вход/выход), rate limits.
- Краевые случаи и как их обрабатываем.

## 6. Данные (Data model)
- Новые/изменённые таблицы, колонки, типы, индексы.
- RLS-политики (кто что видит/пишет).
- Миграции (имя файла в формате `YYYYMMDDHHMMSS_name.sql`).
- Retention / удаление / экспорт (GDPR).

## 7. API / интеграции
- Эндпойнты: метод, путь, вход, выход, коды ошибок.
- Внешние сервисы (Stripe/itsme/Bpost/…): что вызываем, идемпотентность, вебхуки.
- Серверная авторизация (RLS + проверка роли), `supabaseService()` только в server-only.

## 8. Безопасность, приватность, комплаенс
- Угрозы и контрмеры (фрод, ATO, утечки).
- GDPR/DSA/GPSR/Recupel/DAC7 — что применимо.
- Аудит-логи событий.

## 9. Аналитика и события (Telemetry)
- Какие события логируем (имя, свойства).
- Дашборды/воронки, которые из этого строим.

## 10. Доступность и i18n
- Клавиатура, скринридер, контраст, тач-таргеты ≥44px.
- Список i18n-ключей; что нельзя хардкодить.

## 11. Тестирование
- Unit / API / e2e (Playwright) сценарии.
- Что именно проверяем перед релизом (acceptance checklist).

## 12. План внедрения (Rollout)
- Этапы (MVP → улучшения), фиче-флаги.
- Зависимости и порядок.
- Оценка усилий (S/M/L).

## 13. Открытые вопросы и решения
- Что ещё нужно решить (и кто решает).
- Принятые решения с обоснованием (лог).

## 14. Критерии готовности (Definition of Done)
- [ ] Функционально работает по сценариям §3–§5.
- [ ] Данные/RLS/миграции по §6.
- [ ] Тесты по §11 зелёные.
- [ ] a11y/i18n по §10.
- [ ] Аналитика по §9 пишется.
- [ ] Комплаенс по §8 закрыт.
