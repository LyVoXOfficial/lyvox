# LyVoX — заметки live-обзора (capture pack)

Дата: 2026-06-27 · Источник: live www.lyvox.be через Claude in Chrome (desktop, вьюпорт ~1718px; мобильную отрисовку инструмент не эмулирует — мобайл смотрим по коду). Сессия в браузере периодически мигает между гостем и `zviedre2` (SSR→гидрация). Контента мало: 4 тестовых объявления (BMW от одного продавца).

Это сырьё для субагентов. Точные пути кода — в репозитории.

## Сквозные находки (видно на всех страницах)
- **Язык по умолчанию на экране — русский.** Это из-за cookie `locale=ru` в данном браузере, не хардкод (middleware defaultLocale=`en`, выбор по Accept-Language). Но для BE дефолт `en` тоже не идеален — стоит гео/Accept-Language → nl/fr.
- **SEO title одинаковый «LyVoX»** на home/discover/search; специфичный только на карточке объявления.
- **Meta description одинаковый** «Купи-продай, услуги и недвижимость» (RU) почти везде; на карточке = сырое описание объявления (тестовое «test test»).
- **canonical:** отсутствует на home/discover/search; на карточке **есть, но с багом двойного слеша** `https://www.lyvox.be//ad/...`.
- **og:image:** на home — `lyvox.svg` (SVG не рендерится в соц-превью); на карточке — **подписанный Supabase-URL с истекающим токеном** (превью отвалится после экспирации).
- **hreflang** присутствует (en/nl/fr/ru/de) — хорошо. **og:locale=ru** в данном рендере.
- **Нет robots.txt, sitemap.xml, manifest** (проверено в коде/public). Крупный SEO/discoverability-пробел.
- **Шапка мигает** между «Войти/Регистрация» и аккаунтом (auth-state flicker при гидрации) — UX-шум.
- **Каталог категорий медленно грузится** («Загрузка категорий…», «Загрузка…» в фильтрах) на home и search — перцептивная медлительность; кэшировать схему.
- Топбар-полоска с трастовыми тезисами (Сигналы проверенных продавцов / Модерация против мошенничества / Создано для местных сделок) — хорошо для trust-позиционирования.

## Страница: Home (/)
- Hero: «Покупайте и продавайте с людьми, которым доверяете» + подзаголовок про сравнение продавцов и безопасные переписки. Сильный trust-месседж.
- Правая карточка-статистика: 4 свежих / 4 верифицированных / 1 бесплатных + CTA «Подать объявление». На пустом маркете цифры «4/4/1» выглядят слабо (соц-доказательство наоборот вредит). Чипы быстрых категорий (Транспорт/Электроника/Проверенные продавцы/Дом).
- Блок «Свайп-режим» (вход в discover). Три карточки ценности (проверенные продавцы / местные платежи / сообщить о подозрительном). Карусель категорий снизу.
- h1 один (ок). 6 изображений, все с alt.
- UX-флаги: дубль поиска (в шапке и в hero); цифры-статистика подчёркивают пустоту; первый экран desktop почти не показывает товары (нет «витрины» сразу).

## Страница: Discover (/discover)
- Заголовок «Свайп-режим», чипы Drops (Только что / До €50 / Проверенные продавцы / Дешевле) + «Сбросить предпочтения». Кнопки pass/act/like внизу.
- **Дека ПУСТАЯ** — карточек нет, при этом нет видимого empty-state в области деки (просто пусто). Вероятно, все 4 объявления отфильтрованы как «seen» (localStorage). Плохой первый экран: пусто и непонятно.
- Подтверждает PRD 01: нужен явный empty-state, и пересмотр seen-фильтра при малом каталоге.

## Страница: Search (/search?q=BMW)
- Хороший каркас: заголовок «BMW в Бельгии», «Сначала новые» (сортировка), «Сохранить поиск», «Свайп-режим». Слева фильтры: Категория (Загрузка…), Цена (слайдер 0–10000 + поля), тумблер «Только верифицированные», Состояние (Новое/Бу/На запчасти), Сортировка, Локация, Применить/Очистить.
- На момент скрина область результатов — спиннер (медленно), хотя по BMW есть 3 объявления. Категория-фильтр «Загрузка…».
- title=LyVoX, canonical=null. h1=«BMW в Бельгии» (норм, локализован под запрос+страну).
- UX-флаги: медленный first paint результатов; нет видимого счётчика результатов; зависимость от загрузки каталога.

## Страница: Ad detail (/ad/:id) — «BMW 5 Series 1996»
- Сильная trust-панель справа: «Только для подтверждённых», бейджи «Проверен»/«Телефон подтверждён», CTA «Написать продавцу», пояснение «после открывается приватный чат; LyVoX не обрабатывает платежи», «Пожаловаться».
- Галерея фото, бейдж «Верифицированный продавец», «Местоположение не указано», дата размещения, цена 6000€, описание (тестовое), блок «Подтвердите, чтобы увидеть продавца» (identity-gating — соответствует стратегии), блок «Безопасность и доверие».
- title специфичный (хорошо); jsonld×2 (вероятно Product/Breadcrumb).
- **SEO-баги:** canonical с двойным слешем; og:image — подписанный истекающий URL.
- UX-сильно: identity-gating и анти-скам формулировки сделаны грамотно.

## Инфраструктура (из кода)
- **Security headers** (next.config.ts): HSTS, X-Content-Type-Options=nosniff, X-Frame-Options=SAMEORIGIN, Referrer-Policy=strict-origin-when-cross-origin, Permissions-Policy (camera/mic/geo выключены). **CSP — Content-Security-Policy-Report-Only** (НЕ enforced; script-src допускает 'unsafe-inline'/'unsafe-eval'). Promote CSP к enforced после валидации; в будущем — nonce вместо unsafe-inline.
- **Images:** next/image с avif/webp, deviceSizes, remotePatterns только supabase public. (Но карточки в коде местами используют сырой `<img>` — см. PRD 31.)
- **i18n middleware:** locale из ?lang / cookie / Accept-Language; supported en/fr/nl/ru/de; defaultLocale=en.
- **Нет sitemap/robots/manifest.**
