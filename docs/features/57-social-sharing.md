> **Приоритет:** P2
> **Зависит от:** [[42-i18n-localization]] (OG/locale), [[55-referrals]] (реф-параметры)

## 1. Зачем это (Problem & Outcome)
- **Проблема.** Органический рост дёшев, если объявления легко расшарить в WhatsApp/FB/Instagram.
- **Исход.** Кнопка «Поделиться» с красивыми OG-превью; deep-link обратно в LyVoX; (опц.) реф-параметры. Петля роста.
- **KPI:** число шарингов, входящий трафик из шаринга, конверсия пришедших.

## 2. Контекст и текущее состояние (As-is)
- SEO canonical/OG есть (`lib/seo`); кнопки шаринга и генерации share-карточек — нет.

## 3. Пользователи и сценарии
- «Поделиться объявлением другу/в группе»; пришедший — открывает товар и регистрируется.

## 4. UX и поведение (детально)
- Кнопка «Поделиться» на карточке/объявлении: нативный share-sheet (Web Share API), копировать ссылку, WhatsApp/FB/Instagram.
- Красивое OG-превью (фото/цена/заголовок), per-locale.
- (Опц.) share с реф-кодом ([[55-referrals]]).
- Состояния: успешно/скопировано/недоступно.

## 5. Логика и правила
- Web Share API на мобиле, фоллбэк копирование/линки на десктопе.
- UTM/реф-параметры для атрибуции.
- Краевые: приватные/снятые объявления (не шарить), отсутствие фото.

## 6. Данные (Data model)
- Без новых таблиц; (опц.) лог share-событий для аналитики/атрибуции.

## 7. API / интеграции
- OG-метаданные (динамические per объявление/locale); Web Share API; (опц.) генерация share-картинки.

## 8. Безопасность, приватность, комплаенс
- Не раскрывать приватное; уважать видимость объявления; GDPR для атрибуции.

## 9. Аналитика
- Шаринги по каналам, входящий трафик, конверсия, вклад рефералов.

## 10. Доступность и i18n
- Доступная кнопка; i18n `share.*` + OG per-locale (NL/FR/EN).

## 11. Тестирование
- e2e: share-sheet/копирование; OG-превью корректно per-locale; снятое объявление не шарится.

## 12. План внедрения (Rollout)
- **LATER (дёшево):** MVP кнопка+OG; затем реф-параметры/share-картинки. Усилие: S.

## 13. Открытые вопросы и решения
- Генерировать ли кастомные share-изображения. Связка с рефералами на старте.

## 14. Критерии готовности (DoD)
- [ ] Кнопка «Поделиться» (Web Share + фоллбэк); корректные OG per-locale.
- [ ] Атрибуция входящих; i18n.

## Ревью-требования и sign-off (2026-06-28)

> P2-фича, но опирается на **ложную предпосылку «OG готов»** (§2). Переиспользовать нечего — OG надо чинить (P0-фундамент аудита 02). Применить F-фундамент: базовый SEO (metadataBase/og:image PNG/публичные media-URL).

### SEO
- **Исправить §2: «OG готов» → «OG требует фикса».** По аудиту 02: og:image на home = SVG (соцсети не рендерят); на `/ad/*` = подписанный Supabase-URL с истекающим токеном (закэшируется битая картинка). Предусловие: og:image PNG **1200×630** для home; **публичный** media-URL/fallback для `/ad/*`.
- **Per-locale OG (специфицировать, не декларировать §4/§10):** `og:locale` = текущая (`nl_BE`/`fr_BE`/…), `og:locale:alternate` = остальные; локализованные `og:title`/`og:description` (переиспользовать `generateMetadata` из PRD 42). Маппинг `og:locale`: `en→en_US, nl→nl_BE, fr→fr_BE, de→de_DE, ru→ru_RU`.
- **canonical всегда без UTM/ref.** При шаринге `?utm=`/`?ref=` Google не должен индексировать параметрические дубли; ref-атрибуция через cookie/JS, не через индексируемый URL (связать §5).
- **(Опц.) `opengraph-image.tsx` через `next/og`:** 1200×630, шрифт с кириллицей, fallback-PNG при отсутствии фото, заголовок+цена+1 фото, кэш.
- **Каналы:** Twitter `summary_large_image`; WhatsApp/Telegram читают OG (Telegram популярен в RU-диаспоре BE); Pinterest Rich Pins (`product`) — пересечение с Product JSON-LD из PRD 62 (fashion/home SEO-трафик). Учесть в §4.
- **Снятые/приватные объявления не шарятся** (есть в §5 — сохранить); JSON-LD/OG только для `active`.

**Вердикт:** 🔄 — после фикса OG-фундамента (PNG 1200×630, публичные media-URL) и добавления per-locale OG-спеки + canonical без UTM → ✅. Не блокер запуска (P2).
