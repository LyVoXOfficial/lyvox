### Task 1: Canonical base-URL normalization + metadataBase

**Files:**
- Create: `apps/web/src/lib/seo/baseUrl.ts`
- Modify: `apps/web/src/app/layout.tsx`, `apps/web/src/app/ad/[id]/page.tsx`, `apps/web/src/lib/seo/catalog/common.ts`

**Step 1:** Create `lib/seo/baseUrl.ts`:

```typescript
/** Canonical site origin — env value may carry a trailing slash; always normalize. */
export function getBaseUrl(): string {
  const raw =
    process.env.NEXT_PUBLIC_SITE_URL ||
    process.env.NEXT_PUBLIC_BASE_URL ||
    "https://www.lyvox.be";
  return raw.replace(/\/+$/, "");
}

/** Join path onto the base origin, guaranteeing exactly one slash. */
export function absoluteUrl(path: string): string {
  return `${getBaseUrl()}${path.startsWith("/") ? path : `/${path}`}`;
}
```

**Step 2:** In `layout.tsx` `generateMetadata()`: add `metadataBase: new URL(getBaseUrl())`, add `openGraph.url: getBaseUrl()` and `openGraph.siteName: "LyVoX"`.

**Step 3:** In `ad/[id]/page.tsx` and `lib/seo/catalog/common.ts`: find every place that concatenates `NEXT_PUBLIC_SITE_URL`/`BASE_URL` with a path (grep for `SITE_URL`, `BASE_URL`) and route them through `getBaseUrl()`/`absoluteUrl()` so `//ad/...` can never be produced. Do not change the URL shapes otherwise.

**Step 4:** Add unit test `apps/web/src/lib/seo/__tests__/baseUrl.test.ts`: trailing slash stripped, path joined with single slash, default fallback. Run `pnpm test -- --run apps/web/src/lib/seo`.

**Step 5:** `pnpm typecheck` → commit `feat(seo-p0): normalize base URL — kill //canonical + metadataBase`.
