**Do/defer/reject:** PersonalizationMode-контракт — **do** (при старте Store-of-One MVP); dwell-бакетирование — **do сейчас** (маленький фикс); строгий радиус — **do** в волне /search.

## Агент B — Жюльен (Валлонский автолюбитель, запчасти E39)

**Мои находки:**

1. **Ключевое возражение Codex уже закрыто кодом**: критерий «BMW 5 Series year 1996 → ambiguous, not first match» **уже проходит**. `resolveGeneration.ts:1` возвращает `"unique" | "ambiguous" | "none"`, строки 47/58 возвращают `ambiguous` с кандидатами, и существует юнит-тест ровно этого кейса: `resolveGeneration.test.ts:23` — «(a) 1996 BMW 5-Series → ambiguous (E34 ends 1996, E39 starts 1995)». На `/ad` двусмысленность рендерится CTA «укажите поколение» вместо тихого выбора (PRD 62, bug #1996 закрыт). Codex писал требование, не зная, что F7 его уже выполнил.
2. **Маскирование технических ссылок — гап реален, но не такой, как заявлен**: `scrubContacts.ts` НАМЕРЕННО сохраняет IMEI/EAN/серийники (комментарий на строке 38: контiguous digit strings не маскируются — electronics beachhead), т.е. «masking corrupts serials» — неверно. Но `URL_RE` (строка 33, 56) маскирует **все** URL без allowlist — ссылка на каталог запчастей/realoem погибнет. **Подтверждаю в уточнённой форме: нужен доменный allowlist технических ресурсов или маскирование только контактных каналов.**
3. Глубина vehicle-фильтров: подтверждаю — `SCHEMA_EXCLUDED_TYPES` (`SearchFilters.tsx:62`) исключает vehicle из схемных фильтров; кастомный вертикальный блок есть, но контракт не унифицирован (см. кросс-выводы).
4. Перевод чата/объявлений с глоссарием — фичи нет вовсе; гап подтверждён (это часть фичи №6 брифа).

**Do/defer/reject:** generation-safe search — **уже сделано, добавить только фильтр по generation_id в выдачу**; URL-allowlist в скраббере — **do** (маленький, высокая ценность для B и D); глоссарий переводов — **defer** до MVP переводов.

## Do-not-build list (11.8.5)

- LLM-анализ приватного чата для review-детекции — rules-first (согласен с Codex).
- Авто-создание отзывов/авто-рейтинг — никогда без обеих сторон.
- Vector search как замена classic — только hybrid-пилот.
- Professional-buyer disclosure — правовой базы нет.
