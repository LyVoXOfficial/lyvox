## Принципы исполнения (неизменные)

1. Каждая фича — за capability-флагом или в feature-ветке; атомарные коммиты `feat/<id>-<slug>`.
2. `pnpm typecheck && pnpm test && pnpm lint` — зелёные перед merge.
3. ⛔ Никакого money-flow-② кода в прод до закрытия F3.
4. Деньги/получатели — только server-side (F2), RLS = row-security, колонки — через column-scoped GRANT (выученный урок ×3).
5. Trust-копирайт только фактический (DSA Art.30) — никаких «guaranteed safe».
6. 5 локалей в паритете — guard-тест обязателен для новых ключей.
