# T05 — Строгий радиус в /search + секция «вне радиуса»

**Модель:** средняя (gpt high / sonnet-класс). Осторожно: затрагивает RPC-контракт поиска.
**Ветка:** `feat/strict-radius` · **Приоритет:** P1 · **Оценка:** 3 часа.

## Зачем
Краш-тест, агент A: «Geel + 20 км не должен молча подмешивать Брюссель». Сейчас RPC `search_adverts` умеет lat/lng/radius (PostGIS-гард внутри), но UI-фильтр локации — текстовый `ilike` по названию города. Фолбэк за радиус должен быть ЯВНЫМ.

## КРИТИЧЕСКОЕ ограничение
Сигнатура `search_adverts` — 13 параметров, ФИКСИРОВАНА. НЕ добавлять параметры без координированного DROP+CREATE (см. память в `supabase/migrations/20260628120001_search_fts_weights.sql` — заголовок). Для этой задачи параметры НЕ нужны: radius уже в сигнатуре.

## Шаги
1. Изучи: `apps/web/src/app/search/page.tsx` (как строится запрос, какие URL-параметры), `apps/web/src/components/SearchFilters.tsx` (блок локации), `apps/web/src/app/api/search/route.ts`, `apps/web/src/lib/validations/search-schema` (grep).
2. Проверь, есть ли гео-данные городов: таблица `locations` (колонка `point`) — `psql "$SUPABASE_DB_URL" -c "select count(*), count(point) from locations"`. Если `point` пуст у большинства строк — радиус не сработает; тогда СТОП, статус «BLOCKED: locations.point не заполнен», задача переносится (нужен гео-бэкфилл, отдельное задание).
3. Если точки есть: добавь в UI локации выбор радиуса (select: 10/20/50/100 км, дефолт 50 — ключи i18n `search.radius_*` в 5 локалей) и прокинь lat/lng выбранного города + radius_km в существующие параметры RPC.
4. «Вне радиуса»: если результатов в радиусе < 6 — вторым запросом (radius*3) добери и отрисуй ПОД отдельным заголовком `search.outside_radius` («Дальше от вас» — 5 локалей). НЕ смешивать секции.
5. Тесты: unit на построение параметров; существующие search-тесты не сломаны.

## Проверка
- Локально: `curl "http://localhost:3000/search?..."` — страница 200; в UI две секции при малой выдаче.
- `pnpm typecheck && pnpm test && pnpm lint` зелёные.
- Коммит: `feat(search): strict radius with explicit outside-radius section (T05)` — merge --no-ff, push.
- Прод после деплоя: /search отвечает 200, консоль без ошибок.

## Красные линии
- НЕ менять сигнатуру search_adverts. НЕ делать радиус обязательным (поиск без гео работает как раньше).
- i18n-ключи сразу в 5 локалей, иначе guard-тест упадёт.
