# T15 — URL-локали /nl /fr /de /en /ru + hreflang

**Модель:** 🔴 сильная (gpt xhigh / opus-класс) + план + внимательное ревью. Архитектурная миграция маршрутизации.
**Ветка:** `feat/url-locales` · **Приоритет:** P1 (высокая ценность SEO, чем позже — тем дороже) · **Оценка:** 2 дня. Делить на слои, коммит на слой.

## Зачем
Сейчас локаль — из cookie (`apps/web/src/i18n/server.ts:26`), один URL на все языки → Google индексирует ОДНУ версию, «5 языков» (главный дифференциатор, ru — бесконкурентная ниша) в органике не существует. Нужны path-локали + hreflang в статической ISR-форме.

## КРИТИЧЕСКИЕ ловушки (из памяти проекта — прочти AGENTS.md §4-5)
- **ДВА middleware**: `apps/web/middleware.ts` (i18n+session) И `apps/web/src/middleware.ts` (route-protection). В Next активен ОДИН (корень app). Определи какой активен ДО правок — иначе редиректы локалей не сработают или задублируются. Опиши в отчёте, какой активен и почему.
- **hreflang shallow-merge**: `alternates` дочернего роута ЗАМЕЩАЕТ родительский целиком (баг ловили в SEO-волне). Любая per-page alternates должна пере-декларировать `languages`.
- **`?lang=` антипаттерн** фрагментирует full-route cache — эта миграция должна его заменить, а не сосуществовать.
- **Cloudflare** кэширует ~4ч — прод-проверки с cache-buster.

## Скоуп (слои)
1. **Решение о форме** (ОБЯЗАТЕЛЬНО в отчёте до кода): префикс-роутинг `/[locale]/...` vs domain vs cookie+rewrite. Рекомендация: сегмент `/[locale]` с `en` как default без префикса ИЛИ с префиксом — выбери и обоснуй с учётом 159+ уже проиндексированных бесслаг-URL (не сломать редиректами). Middleware: один автодетект-редирект (307, кэшируемый) на голом `/`, дальше пользователь живёт на path-URL без повторных редиректов.
2. **Рендеринг**: HTML каждой локали по URL (SSR/ISR), НИКАКОГО client-side locale swap (флеш чужого языка + CLS). `getI18nProps`/`resolveLocale` берут локаль из сегмента пути, не из cookie.
3. **hreflang + canonical**: на всех индексируемых страницах (`/`, `/c/*`, `/ad/*`, `/sell`, `/guides`) — `alternates.languages` со всеми 5 + `x-default`, canonical на текущую локаль. Через `getBaseUrl()`/`absoluteUrl()` (существующие).
4. **sitemap**: множить URL по локалям с hreflang-аннотациями (или отдельные sitemap на локаль).
5. **Переключатель языка**: one-tap, сохраняет ТЕКУЩУЮ страницу и скролл (не сбрасывает на главную) — `LanguageSwitcher.tsx`.

## Проверка
- Полный сьют; локально каждая локаль рендерит свой HTML по URL (curl `/nl/...`, `/fr/...` — правильный язык в SSR).
- hreflang присутствует и симметричен (все 5 + x-default) на ключевых страницах.
- merge --no-ff, push. Прод: `curl` каждой локали с cache-buster; проверить, что старые бесслаг-URL редиректят/канонятся, а не 404.

## Красные линии
- Не сломать 159+ проиндексированных URL (редиректы/canonical, не 404). Один автодетект-редирект максимум. Никакого client locale-swap. Паритет модерации/правил на всех локалях (DSA). Если объём слишком большой для одного захода — раздели на «инфраструктура локалей» и «hreflang+sitemap» два merge.
