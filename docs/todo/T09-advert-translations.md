# T09 — Переводы объявлений: миграция + async-джоб + лейблы

**Модель:** сильная (gpt xhigh / opus-класс). Многослойная фича: БД + джоб + UI + SEO-ограничения.
**Ветка:** `feat/advert-translations` · **Приоритет:** P1 (высший ROI по вердикту CTO-брифа §6) · **Оценка:** 2 дня. Делить на коммиты по слоям.

## Зачем
Один листинг продавца должен быть находим на nl/fr/en/de/ru. Схема и правила — в брифе `docs/strategy/AI_FEATURES_2026_EVALUATION_FOR_FABLE5.md` §6 (обязательно прочитать целиком) + ревью-состояния из §11.6.

## Слои (коммит на слой)
1. **Миграция** `advert_translations` — взять SQL из брифа §6 ДОСЛОВНО, добавить: RLS (SELECT публичный для status='published'; INSERT/UPDATE только service_role), status расширить значением `'reviewed'` (после 'published'), индекс `(advert_id, target_locale)`. Идемпотентно. Применить db push, проверить psql'ом.
2. **Провайдер-абстракция** `apps/web/src/lib/translations/provider.ts`: интерфейс `translate(text, from, to): Promise<string>` + ДВЕ реализации: `NoopProvider` (возвращает null → джоб помечает недоступность) и заглушка `ExternalProvider` за env `TRANSLATION_PROVIDER_URL/KEY` (fetch, таймаут 10с). Капабилити-флаг `CAPABILITY_ADVERT_TRANSLATIONS` в `lib/capabilities.ts` (по образцу существующих) — ВСЁ за флагом, дефолт OFF.
3. **Джоб**: route `GET /api/cron/translate-adverts` по образцу `api/cron/business-verify` (Bearer CRON_SECRET, fail-closed): берёт до 20 активных объявлений без переводов (или со stale), для каждого — 4 целевые локали (все кроме source_locale из `adverts.content_locale`), пишет status='draft'→'published' при успехе. source_hash = sha256(title+description). При правке источника (найди PATCH advert route) — пометить существующие переводы 'stale' (обновление одним UPDATE).
   **Seed-развилка (решение основателя на момент активации, зашить переключателем):** env `TRANSLATE_INCLUDE_SEED` (default `true`). При `true` джоб переводит и seed-витрину (демонстрирует мультиязычность, стоимость копеечная); при `false` — пропускает объявления seed-аккаунтов (владельцы с email `%@lyvox-seed.be` — join на auth-таблицу через service-клиент). Пока флаг фичи OFF, ничего не переводится вообще — развилка актуальна только при активации.
4. **UI на /ad**: если локаль зрителя ≠ content_locale и есть published-перевод — показать переведённые title/description + бейдж `advert.translation_generated` («Переведено автоматически с {язык}») + тумблер `advert.translation_show_original`. Ключи в 5 локалей. Оригинал ВСЕГДА доступен одним тапом. JSON-LD продолжает использовать ОРИГИНАЛ (не переводы) — никакой индексации переводов в этой волне.
5. **Тесты**: RLS-негатив (anon не видит draft), source_hash-инвалидация, cron fail-closed без секрета, UI-фолбэк на оригинал при отсутствии перевода.

## Проверка

- Коммиты по слоям, merge --no-ff, push.

## Красные линии (из брифа §6, дословно)
- Никогда не скрывать, что перевод машинный. Никогда не заменять оригинал. Юр-тексты платформы через этот пайплайн НЕ переводить. Не «усиливать» заявления продавца. НЕ индексировать переводы (hreflang/canonical-контроль — отдельная будущая волна).
- Модерация: переводы не обходят prohibited-content проверки источника.
