# T18 (LAUNCH-GATE) — Seed-выключатель

**Модель:** 🔴 сильная (gpt xhigh / opus-класс) + ревью. Затрагивает агрегаты, JSON-LD, RLS.
**Ветка:** `feat/seed-switch` · **Приоритет:** launch-gate (последний кодовый шаг перед аудиторией) · **Оценка:** 1 день.

## Зачем и ГРАНИЦА (решение основателя)
Seed-контент — намеренная витрина ДО запуска (см. `docs/strategy/SITE_BLUEPRINT.md` трейд-оффы + память seed-system). **Это задание строит ТОЛЬКО выключатель** — механизм исключения seed из агрегатов/социального-пруфа/JSON-LD/медиан, чтобы к запуску переключение было одним действием. **Сам purge seed НЕ выполнять** — он делается ОТДЕЛЬНО, только по явной команде основателя перед рекламой.

## Текущий факт (проверено)
Seed-аккаунты маркируются ТОЛЬКО доменом email `@lyvox-seed.be` (в `auth.users.email`). Колонки `is_seed` НЕТ. `top_sellers` — отдельная таблица/view (`api/top-sellers/route.ts` читает `from("top_sellers")`) без фильтра seed.

## Скоуп
1. **Флаг в данных** (миграция idempotent): добавить `profiles.is_seed boolean not null default false`; backfill `update profiles set is_seed = true where id in (select id from auth.users where email like '%@lyvox-seed.be')`. Это устойчивее, чем join на auth-email в каждом запросе. Триггер/примечание: новые seed-пользователи сидера должны выставлять флаг (обнови `scripts/seed/seed.mjs` — при создании профиля ставить is_seed=true; НЕ запускать сидер, только код).
2. **Единый гейт исключения** — env/капабилити `EXCLUDE_SEED_FROM_AGGREGATES` (default `false` сейчас = витрина видна; основатель ставит `true` на запуске). При `true` исключать `is_seed` продавцов/объявления из:
   - `top_sellers` (route + сам источник таблицы/view — если materialized view, добавить фильтр в его definition-миграцию);
   - медиан/оценщика цен (`estimate_price` RPC из T06 — добавить `and p.is_seed = false` за условием, управляемым параметром/GUC или отдельной функцией; см. TODO-коммент, который T06 оставил);
   - JSON-LD агрегатов (AggregateRating, ItemList) — исключать seed из подсчётов;
   - счётчиков активности/«N просмотров» если считаются от аккаунтов.
   Важно: при `false` — поведение БАЙТ-в-БАЙТ текущее (витрина как есть).
3. **НЕ трогать**: сами seed-объявления/аккаунты не удалять, не прятать с выдачи/категорий (они остаются витриной). Выключатель влияет ТОЛЬКО на агрегаты/social-proof/структурированные данные.

## Проверка
- Миграция применена (db push + psql: `select count(*) from profiles where is_seed`); при флаге OFF агрегаты не изменились (сравни top-sellers ответ до/после).
- Полный сьют; тест на функцию исключения (seed продавец не попадает в агрегат при флаге ON).
- merge --no-ff, push.

## Красные линии
- Purge НЕ выполнять — только механизм. Флаг default OFF (витрина сохраняется). Миграция idempotent, RLS не ослаблять. При OFF — ноль изменений в проде. Не удалять и не деиндексировать seed-объявления.
