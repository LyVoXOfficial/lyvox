# T14 — Системный проход empty-states + /search полировка

**Модель:** 🟡 средняя (gpt high / sonnet-класс).
**Ветка:** `feat/empty-states-search` · **Приоритет:** P1 · **Оценка:** 1 день.

## Зачем
SITE_BLUEPRINT волна 4 + аудит: на малой ликвидности пустой экран — самый частый, он должен предлагать живой путь, а не «0 найдено» тоном приговора. Часть уже есть — довести и унифицировать.

## Что УЖЕ есть
- Skeleton: `apps/web/src/components/marketplace-grid-states.tsx` (`AdsGridSkeleton`), уже используется в DiscoveryFeed.
- Поиск: `apps/web/src/app/search/page.tsx`, `apps/web/src/components/SearchFilters.tsx`, валидация `lib/validations/search*`.
- Zero-result частично: проверь grep'ом `no_results`/`SaveSearch` в search page — что уже показывается.
- SaveSearch: `apps/web/src/components/saved/SaveSearchButton.tsx`.

## Скоуп
1. **Единое правило empty-state** (вынеси в один компонент `components/EmptyState.tsx` если ещё нет): заголовок + подсказка + основной CTA. Применить на: `/search` zero-result, `/c/*` пустая/тонкая категория, `/discover` исчерпание колоды, `/saved` пусто, `/favorites` пусто. Каждый empty НИКОГДА не «тут ничего нет» — всегда действие (расширить/сохранить/сбросить фильтр).
2. **Zero-result авторасширение на /search**: при 0 точных совпадений — честный текст «Точных совпадений нет — похожее рядом» + попытка ослабить фильтры (снять часть фильтров / расширить, с подсчётом «+N объявлений»), затем SaveSearch как главный CTA («Узнайте первым, когда появится»). НЕ выдумывать результаты — только реально существующие из ослабленного запроса.
3. **Skeleton вместо спиннера** везде в /search и /c/* (размеры точно под сетку aspect 4:3 → CLS=0).
4. **verified-чип с предпросмотром счётчика**: тумблер «Только верифицированные» поднять в чипы над выдачей с предпросмотром «покажем N из M» ДО применения (prefetch count) + tooltip содержания проверки (email+phone; бизнес +KBO/VIES). Логировать в `analytics_events` (F6 — см. `lib/analytics`).
5. **Единая модель фильтров**: на десктопе убрать двойственность «мгновенно + Применить» — оставить debounce 300мс; в мобильном drawer на кнопке «Применить» — живой счётчик «Показать N».

## Проверка
- Focused search-тесты + полный сьют. Локально: пустой поиск показывает путь, не тупик.
- i18n новых строк — 5 локалей. merge --no-ff, push, прод /search 200.

## Красные линии
- Не показывать выдуманные результаты. `/search` остаётся `noindex` (не трогать robots-мету). Счётчики только реальные (порог для «N из M» — из реальных данных). i18n 5 локалей.
