# T13 — UX1: редизайн-фиксы страниц по мокапам

**Модель:** 🟡 средняя (gpt high / sonnet-класс). Осторожно: presentation-only, много мелких правок.
**Ветка:** `feat/ux1-redesign-fixes` · **Приоритет:** P1 · **Оценка:** 1-2 дня. Делить по страницам, коммит на страницу.

## Зачем
UX-психологический аудит `docs/features/audit/01-ux-psychology-audit.md` содержит конкретные находки по страницам (home/discover/search/ad/post/profile). Мокапы-референс: `docs/features/audit/mockups/redesign-mockups.html` и `category-detail-mockups.html` (открой в браузере/прочитай разметку как ориентир, НЕ как пиксель-контракт).

## КРИТИЧЕСКОЕ — audit-first + presentation-only
1. СНАЧАЛА прочитай `01-ux-psychology-audit.md` целиком и выпиши в отчёт-заметку (`docs/todo/notes/T13-scope.md`, отдельный коммит) список конкретных находок, которые ещё НЕ закрыты предыдущими волнами. МНОГОЕ уже сделано в этой сессии — не переделывай:
   - первый экран главной (stats-карточка убрана, один поиск, товар-first) — Волна 1, done;
   - мобильная раскладка/тач-таргеты/крупный шрифт — UX2 + S23U, done;
   - можибейк локалей — done; фид-кап — done; share-баннер — done.
   Бери ТОЛЬКО незакрытые находки. Дубли с уже сделанным — вычёркивай в заметке с пометкой «done in <волна>».
2. Каждая правка — **presentation-only**: разметка/Tailwind/анимации. НЕ трогать data-fetching, props, гейты, RLS, роуты, i18n-ключи (только если аудит требует новую видимую строку — тогда 5 локалей).
3. Не «редизайнить ради редизайна» — чинить конкретную названную в аудите проблему (иерархия, доверие, когнитивная нагрузка, пустые состояния). Мокап — ориентир, не обязанность копировать пиксель.
4. Trust-копирайт только фактический (DSA Art.30) — никаких «guaranteed safe».

## Шаги
- Заметка со scope (коммит 1).
- По странице за коммит: `feat(ux1): <страница> — <что чинит из аудита>`. Проверяй визуально через preview-сервер (curl SSR + при возможности скриншот), помни про stale-CSS дев-сервера (§5 AGENTS.md — читай SSR-HTML, не кэш браузера).
- Мотион — по фреймворку emil-design-eng (если добавляешь): редкие поверхности можно оживить, частые — нет; ≤300мс, ease-out, prefers-reduced-motion.

## Проверка
- `pnpm typecheck && pnpm test && pnpm lint` зелёные.
- Каждая страница проверена в preview (SSR + скриншот в заметку).
- merge --no-ff, push, прод-проверка ключевых страниц (curl 200 + cache-buster).

## Красные линии
- Presentation-only: ноль изменений логики/данных/гейтов. Новые видимые строки — 5 локалей. Никаких фейковых trust-заявлений. Не ломать identity-гейты физ-продавца (redact на server-boundary) и мобильные bottom-spacing переменные (--bottom-nav-h).
