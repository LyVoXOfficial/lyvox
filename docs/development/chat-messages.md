# Чат / Сообщения

## Architecture

См. также: `../domains/chat.md`

**Технологии:**
- Realtime: Supabase Realtime (websocket)
- SSR: начальная загрузка истории
- Client subscription: live updates

## Database Schema

```sql
CREATE TABLE public.conversations (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  created_by uuid NOT NULL REFERENCES auth.users(id),
  advert_id uuid REFERENCES public.adverts(id),
  last_message_at timestamptz,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE TABLE public.conversation_participants (
  conversation_id uuid REFERENCES public.conversations(id) ON DELETE CASCADE,
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  role text CHECK (role IN ('owner', 'peer')),
  muted boolean DEFAULT false,
  last_read_at timestamptz,
  PRIMARY KEY (conversation_id, user_id)
);

CREATE TABLE public.messages (
  id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  conversation_id uuid NOT NULL REFERENCES public.conversations(id) ON DELETE CASCADE,
  author_id uuid NOT NULL REFERENCES auth.users(id),
  body text NOT NULL,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE INDEX idx_messages_conversation_created 
  ON public.messages(conversation_id, created_at DESC);
CREATE INDEX idx_messages_author 
  ON public.messages(author_id, created_at DESC);
CREATE INDEX idx_conversations_last_message 
  ON public.conversations(last_message_at DESC);
```

## RLS Policies

**conversations:**
```sql
-- Участники могут читать свои диалоги
CREATE POLICY "participants_can_read" 
  ON public.conversations FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM public.conversation_participants
      WHERE conversation_id = conversations.id
        AND user_id = auth.uid()
    )
  );

-- Админы могут читать все
CREATE POLICY "admins_can_read_all"
  ON public.conversations FOR SELECT
  USING (public.is_admin());
```

**messages:**
```sql
-- Участники диалога могут читать сообщения
CREATE POLICY "participants_can_read_messages"
  ON public.messages FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM public.conversation_participants
      WHERE conversation_id = messages.conversation_id
        AND user_id = auth.uid()
    )
  );

-- Только автор может писать
CREATE POLICY "author_can_insert"
  ON public.messages FOR INSERT
  WITH CHECK (author_id = auth.uid());
```

## API Endpoints

| Endpoint | Метод | Описание | Auth |
|----------|-------|----------|------|
| `/api/chat/start` | POST | Создать/найти диалог | Required |
| `/api/chat/send` | POST | Отправить сообщение | Required |
| `/api/chat/history` | GET | История с пагинацией | Required |
| `/api/chat/read` | POST | Отметить прочитанным | Required |

**POST /api/chat/start:**
```typescript
Body: {
  advert_id?: uuid,
  peer_id: uuid
}

// Логика:
// 1. Проверка что peer_id != auth.uid()
// 2. Поиск существующего диалога по advert_id + participants
// 3. Если не найден - создание нового
// 4. Возврат conversation_id
```

**POST /api/chat/send:**
```typescript
Body: {
  conversation_id: uuid,
  body: string
}

// Логика:
// 1. Проверка участия в диалоге
// 2. Вставка сообщения
// 3. Обновление last_message_at в conversations
// 4. Rate limiting: 20 сообщений/мин
```

**GET /api/chat/history:**
```typescript
Query: {
  conversationId: uuid,
  cursor?: bigint, // id последнего загруженного сообщения
  limit?: number // default 50
}

// Возвращает сообщения до cursor (более старые)
```

## Realtime Subscription

**Hook: `useRealtimeMessages`**
```typescript
function useRealtimeMessages(conversationId: string) {
  const [messages, setMessages] = useState<Message[]>([]);
  const supabase = supabaseBrowser();

  useEffect(() => {
    const channel = supabase
      .channel(`conversation:${conversationId}`)
      .on('postgres_changes', {
        event: 'INSERT',
        schema: 'public',
        table: 'messages',
        filter: `conversation_id=eq.${conversationId}`
      }, (payload) => {
        setMessages(prev => [...prev, payload.new as Message]);
      })
      .on('system', {}, (payload) => {
        // Handle errors, reconnect logic
      })
      .subscribe();

    return () => {
      channel.unsubscribe();
    };
  }, [conversationId]);

  return messages;
}
```

**Reconnect logic:**
- Экспоненциальная задержка при ошибках
- Автоматическая переподписка
- Индикатор "Reconnecting..."

## UI Features

### Список диалогов

**Отображение:**
- Аватар собеседника
- Имя собеседника
- Последнее сообщение (preview)
- Время последнего сообщения
- Индикатор непрочитанных (badge)

**Сортировка:**
- По `last_message_at` DESC (самые свежие сверху)

### Окно чата

**Функции:**
- Виртуализированный список сообщений (react-window)
- Оптимистичная отправка (показывать сразу, потом подтвердить)
- Retry при ошибке отправки
- Индикатор "печатает..." (future, через presence)
- Timestamps (относительные: "2 мин назад")
- Read receipts (future)

## Rate Limiting

**Ограничения:**
- 20 сообщений / минуту на пользователя
- Проверка в `/api/chat/send`

**Response при превышении:**
```json
{
  "ok": false,
  "error": "rate_limited",
  "retry_after_seconds": 15
}
```

## Уведомления

**Триггеры:**
- Новое сообщение (если участник offline)
- Email уведомление (если preferences разрешают)
- Push уведомление (future)

**Дедuпликация:**
- Не отправлять уведомления если уже отправлено за последние 60 секунд
