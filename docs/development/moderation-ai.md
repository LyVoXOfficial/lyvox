## MVP Enhancements

### AI Moderation Flow

| Шаг | Действие | Результат |
|-----|----------|-----------|
| 1 | Пользователь публикует объявление | `status='draft'` → `'pending_review'` (если премодерация включена) |
| 2 | AI анализ объявления | Оценка риска: 0-100 |
| 3 | Автоматическое решение | score < 30: auto-approve<br>score 30-70: manual review<br>score > 70: auto-reject |
| 4 | Уведомление пользователю | Одобрено / Отклонено / Требуется проверка |

### AI Scoring Criteria

| Критерий | Вес | Описание |
|----------|-----|----------|
| Текст (spam/fraud keywords) | 40% | Проверка на запрещенные слова, подозрительные фразы |
| Изображения (NSFW/качество) | 30% | Анализ фото на NSFW контент, качество изображений |
| Цена (аномальность) | 15% | Слишком низкая цена относительно категории |
| Поведение продавца | 15% | Trust score, история нарушений |

### Database Changes

```sql
ALTER TABLE public.adverts ADD COLUMN ai_moderation_score integer;
ALTER TABLE public.adverts ADD COLUMN ai_moderation_reason text;
ALTER TABLE public.adverts ADD COLUMN moderation_status text DEFAULT 'pending'; 
-- 'pending', 'approved', 'rejected', 'auto_approved', 'auto_rejected'

CREATE TABLE public.moderation_logs (
  id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  advert_id uuid NOT NULL REFERENCES public.adverts(id),
  moderator_id uuid REFERENCES auth.users(id), -- NULL if AI
  action text NOT NULL, -- 'approve', 'reject', 'auto_approve', 'auto_reject'
  reason text,
  ai_score integer,
  created_at timestamptz DEFAULT now()
);

CREATE INDEX idx_moderation_logs_advert 
  ON public.moderation_logs(advert_id, created_at DESC);
```

## API Endpoints

| Endpoint | Метод | Описание | Auth |
|----------|-------|----------|------|
| `/api/moderation/analyze` | POST | AI анализ объявления | Required |
| `/api/moderation/review` | POST | Решение модератора | Admin |
| `/api/moderation/queue` | GET | Очередь модерации | Admin |

**POST /api/moderation/analyze:**
```typescript
Body: {
  advertId: uuid
}

Response: {
  ok: true,
  score: 45,
  reason: "Price seems unusually low for this category",
  autoAction: 'manual' // 'approve' | 'reject' | 'manual'
}
```

**POST /api/moderation/review:**
```typescript
Body: {
  advertId: uuid,
  action: 'approve' | 'reject',
  reason?: string
}

// Логика:
// 1. Обновление moderation_status
// 2. Если approve → status='active'
// 3. Если reject → status='blocked'
// 4. Логирование в moderation_logs
// 5. Уведомление пользователю
```

## AI Implementation

### Edge Function

**Путь:** `supabase/functions/ai-moderation/index.ts`

**Input:**
```typescript
{
  title: string,
  description: string,
  price: number,
  category: string,
  images: string[], // URLs
  seller_trust_score: number
}
```

**LLM Prompt:**
```
Analyze this marketplace listing for spam, fraud, or policy violations.

Title: {title}
Description: {description}
Price: {price} EUR
Category: {category}

Score criteria:
- Text analysis (spam/fraud keywords): 0-40 points
- Image analysis (NSFW/quality): 0-30 points
- Price analysis (suspiciously low): 0-15 points
- Seller behavior (trust score, history): 0-15 points

Return JSON:
{
  "score": 0-100,
  "reason": "explanation",
  "autoAction": "approve" | "reject" | "manual"
}
```

**Output:**
```typescript
{
  score: number, // 0-100
  reason: string,
  autoAction: 'approve' | 'reject' | 'manual'
}
```

### LLM Provider

**Опции:**
- OpenAI GPT-4 (recommended)
- Anthropic Claude
- OpenAI GPT-3.5 (fallback, дешевле)

**Cost optimization:**
- Использовать GPT-3.5 для низкого риска объявлений
- GPT-4 для спорных случаев

## Moderation Queue

**Фильтры:**
- Status: pending / auto_approved / auto_rejected
- AI Score: диапазон (например, 30-70)
- Дата создания
- Категория

**UI:**
- Список объявлений с AI score
- Быстрые действия: Approve / Reject
- Детальный просмотр объявления

## Automatic Actions

**Auto-approve (score < 30):**
1. Установить `moderation_status = 'auto_approved'`
2. Установить `status = 'active'`
3. Логировать в `moderation_logs`
4. Уведомить пользователя

**Auto-reject (score > 70):**
1. Установить `moderation_status = 'auto_rejected'`
2. Установить `status = 'blocked'`
3. Логировать в `moderation_logs`
4. Уведомить пользователя с объяснением

**Manual review (score 30-70):**
1. Установить `moderation_status = 'pending'`
2. Добавить в очередь модераторам
3. Уведомить пользователя о проверке
